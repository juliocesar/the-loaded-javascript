Stylistic & logic modularity

in modern web apps

‟Complete separation between parts of the UI, looks and functionality-wise. And each still works on it's own.”

— Me

Why?

Easy to test. Easy to fix. Easy to maintain.

How? (styles)

Use a preprocessor. If for no other reason, for this.

Minimal set of superclasses with common properties.

Each CSS/SASS/Stylus file responsible for a UI module.

Cascade as little as you can.

How? (logic)

Implied, but write SMALL modules.

Tie modules together with a common event bus.

Modules use it to broadcast messages interested modules listen to.

Ask yourself these:

How easy it is to test my components separately?

Do my components error out when ran separately?

Do my components look right when rendered individually?

Refactor until you get the right answers to all 3.
