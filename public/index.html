Some base info:

Scripts are loaded two at a time.

Loading scripts blocks the entire UI from rendering.

(Hence loading them at the bottom of the page)

(But that only helps you in a page scenario, not an app)

“But there’s gotta be a better way!”

… I hear you say

Enter defer/async

=

<script> tag attributes that modify the way it loads

<script src="…" defer>

defer automates a task which people have been doing since the 90s:

$(document).ready(function() { … });

It’ll load the JS, but *defer* its execution until the DOM is ready.

<script src="…" async>

async is defer’s dumb cousin

It’ll fire execution as soon as it's loaded

Handy for stuff such as analytics code
